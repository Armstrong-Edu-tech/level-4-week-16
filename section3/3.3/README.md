## Topic 3.3 – User API Testing with Jest Coverage

This folder is part of **Armstrong Level 4 – Week 16: Unit Testing with Jest**, demonstrating **Express.js route testing** and **Jest coverage reporting**.

### What it does

- `app.js`:
  - Configures an Express app with `/users` routes.
  - Exports `{ app }` so Jest + Supertest can send requests without starting the server as a separate process.
- User-related files (`config/db.js`, `models/user.model.js`, `controllers/user.controller.js`, `routes/user.routes.js`) define the API.
- `__tests__/user.test.js`:
  - Contains Jest + Supertest tests for the user endpoints.
  - Asserts on status codes and response bodies.
- `coverage/`:
  - Generated by Jest when running tests with coverage enabled.
  - Includes `lcov-report/` HTML pages that show which lines and branches are covered.

### Install & Run

From this folder:

```bash
npm install
npm test        # may already run with coverage, depending on package.json
```

To view coverage, open:

- `coverage/lcov-report/index.html`

in your browser.
